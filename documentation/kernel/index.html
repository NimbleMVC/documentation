Kernel

Kernel to główny komponent systemu, który koordynuje ogólne operacje aplikacji, takie jak obsługa żądań, routing, inicjalizacja konfiguracji i zarządzanie połączeniami z bazą danych. Stanowi centralny punkt zarządzania przepływem aplikacji.

Namespace

Klasa:

Nimblephp\framework\Kernel

Interface:

Nimblephp\framework\Interfaces\KernelInterface

Zmienne

  • public static string $projectPath: Podstawowa ścieżka katalogu projektu.
  • protected RouteInterface $router: Instancja routera obsługującego routing żądania.
  • protected RequestInterface $request: Instancja zarządzająca żądaniami.
  • protected ResponseInterface $response: Instancja zarządzająca odpowiedziami.
  • protected MiddlewareInterface $middleware: Instancja middleware.

Metody

Konstruktor

public function __construct(RouteInterface $router)

Inicjalizuje Kernel z podanym routerem, żądaniem i odpowiedzią. Ustawia także podstawową ścieżkę projektu.

getProjectPath

protected function getProjectPath(): string

Zwraca rzeczywistą ścieżkę katalogu projektu.

handle

public function handle(): void

Uruchamia aplikację, wykonując kolejno metodę bootstrap oraz loadController. Obsługuje także wyjątki zgłaszane w tych metodach.

loadConfiguration

public function loadConfiguration(): void

Ładuje konfigurację z plików .env, obsługując zarówno domyślną konfigurację jak i specyficzną dla środowiska.

bootstrap

protected function bootstrap(): void

Inicjuje sesję, łączy z bazą danych i wykonuje auto-loader klas.

errorCatcher

protected function errorCatcher(): void

Metoda ma wyłapywać i zapisywać do logów błędy

autoCreator

protected function autoCreator(): void

Tworzy niezbędne katalogi w projekcie, jeśli nie istnieją.

initializeSession

protected function initializeSession(): void

Inicjalizuje sesję użytkownika.

connectToDatabase

protected function connectToDatabase(): void

Łączy z bazą danych, używając konfiguracji z plików .env.

autoloader

protected function autoloader(): void

Auto-loader dla klas projektu, ładuje klasy na podstawie namespace.

loadController

protected function loadController(): void

Ładuje i wykonuje kontroler na podstawie danych z routera.

handleException

protected function handleException(Throwable $exception): void

Obsługuje wyjątki zgłaszane podczas działania aplikacji, logując je i w zależności od konfiguracji, wyświetlając informacje o błędzie.

debug

protected function debug(): void

Ustawienia dotyczące wyświetlania błędów, jeśli opcja debugowania jest włączona.

loadModules

protected function loadModules(): void

Załadowywanie oraz początkowa inicjalizacja modułów
